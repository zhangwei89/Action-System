# 行动系统

> **多数人改变不了现状，不是能力不足，而是缺少持续执行的方法。**

互联网时代从不缺知识，缺的是把知识转化为结果的能力。本书聚焦行动系统的构建，从目标设定、计划执行、反馈复盘到长期迭代，帮助读者建立一套能够持续产生结果的行动机制。

**在线阅读**：[https://zhangwei89.github.io/Action-System/](https://zhangwei89.github.io/Action-System/)

---

## 这本书在解决什么问题

多数人不是没有方法，而是：

- 方法之间互相打架，不知道今天该做哪个。
- 方法和生活脱节，回到现实一天根本用不上。
- 没有把方法变成自己的系统，学了一堆理论一切还是老样子。

本书不讲新道理，陪你做一件事：**把那些零散的、互相打架的道理，组装成一套你自己能跑起来的系统。**

## 章节内容

每一章按五个模块组织：

- **核心观点**：本章最关键的一句话，能让你在五分钟内抓住精髓。
- **典型案例**：真实场景的还原，包含人物、冲突、转折、结果。
- **操作步骤**：可以直接照着做的步骤，不超过五条。
- **行动清单**：本章读完，你应该做的三到五件事。
- **复盘问题**：五个问题，帮你把这章的内容和自己的生活连接起来。

这套模块的目的是：**让你读完每一章，都比读之前多做了点什么。**

## 怎么读这本书

**第一种：通读法。** 按顺序读完，每章结尾的"行动清单"都做一遍。适合愿意花一个月系统升级自己的人。

**第二种：问题驱动法。** 先看目录，挑你现在最卡的那个问题对应的章节，读完立刻动手。

**第三种：工具书用法。** 把书放在手边，遇到具体问题再去查。

## 本书特色

- ✅ 不写鸡汤，不喊口号。
- ✅ 不讲大道理，每章都给可执行步骤。
- ✅ 不堆方法，每个工具都标明适用边界。
- ✅ 不写空案例，所有案例都是真实场景的还原。
- ✅ 不追求完美，本身就是一套不完美但能持续运转的方案。
